# NODE API
Обмен клиентов с нодами может происходить в нескольких вариантах
- REST
- websocket

## user
создание и обновление пользователя происходит по инициативе самого пользователя
### api методы
POST /user
- создает пользвателя
GET /user/<id>
- возвращает данные о пользователе
PUT /user/<id>
- обновляет данные о полользвателе

### session
отправка, прием и получение сообщений в рамках сессии
сессии могут быть как direct - по user_id, так и общего типа, по id_сессии
обмен сообщениями в рамках сессии может быть по REST_API так и по websocket

### api методы
POST /session/<sid>
- публикация сообщения в сессию
GET /session/<sid>
- получение сообщений из сессии (с возможностью ограничения времени выборки)
### websocket методы
post, get, subscribe (для автоматического получения новых сообщений в сессии)
